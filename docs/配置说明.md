# 配置文件说明

## 文件说明

| 文件 | 用途 | Git状态 |
|------|------|---------|
| `config.example.json` | 配置模板，供参考 | ✅ 推送到GitHub |
| `config.json` | 用户实际配置 | ❌ 不推送（含敏感路径） |

**新用户操作**：
```bash
# 1. 复制模板
cp config.example.json config.json

# 2. 修改配置（见下文）
# 编辑 config.json，替换为您自己的路径
```

---

## 配置项说明

### 1. 项目路径 (`paths`)

```json
{
  "paths": {
    "project_root": null,        // 项目根目录，null自动检测
    "settings_dir": "设定",       // 设定目录
    "techniques_dir": "创作技法", // 技法目录
    "content_dir": "正文",        // 正文目录
    ...
  }
}
```

**路径格式**：
- 相对路径：相对于 `project_root`，如 `"设定"`
- 绝对路径：如 `"D:/我的项目/设定"` 或 `"E:\\小说资源"`
- 自动检测：设为 `null` 自动检测项目根目录

---

### 2. 数据库配置 (`database`)

```json
{
  "database": {
    "qdrant_host": "localhost",  // Qdrant服务器地址
    "qdrant_port": 6333,          // Qdrant端口
    "collections": {
      "novel_settings": "novel_settings_v2",
      "writing_techniques": "writing_techniques_v2",
      "case_library": "case_library_v2"
    }
  }
}
```

**启动Qdrant（Docker）**：
```bash
docker run -d --name qdrant-server -p 6333:6333 qdrant/qdrant
```

---

### 3. 模型配置 (`model`)

```json
{
  "model": {
    "embedding_model": "BAAI/bge-m3",
    "model_path": null,          // 本地模型路径，null自动检测
    "hf_cache_dir": null,        // HuggingFace缓存目录
    "vector_size": 1024
  }
}
```

**模型路径示例**：
```json
// Windows
"model_path": "E:/huggingface_cache/hub/models--BAAI--bge-m3/snapshots/5617a9f..."

// Linux
"model_path": "/home/user/.cache/huggingface/hub/models--BAAI--bge-m3/snapshots/5617a9f..."
```

**自动检测顺序**：
1. 环境变量 `BGE_M3_MODEL_PATH`
2. 配置文件 `model_path`
3. 常见位置：`~/.cache/huggingface/...`

---

### 4. 小说资源目录 (`novel_sources`)

```json
{
  "novel_sources": {
    "directories": [
      "E:\\小说资源",        // Windows绝对路径
      "/data/novels",        // Linux绝对路径
      "./novels"             // 相对路径
    ]
  }
}
```

**用途**：
- 案例提取时扫描这些目录
- 支持 `txt`, `epub`, `mobi` 格式

**使用**：
```bash
# 扫描小说资源
python tools/case_builder.py --scan

# 或命令行指定（覆盖配置）
python tools/case_builder.py --source-dirs "D:/我的小说"
```

## 常见问题

### Q: 路径应该用哪种格式？

**推荐**：统一使用正斜杠 `/` 或双反斜杠 `\\`

```json
// ✅ 推荐
"path": "D:/小说/设定"
"path": "D:\\小说\\设定"

// ❌ 不推荐（单反斜杠会转义）
"path": "D:\小说\设定"   // \u 会变成特殊字符
```

### Q: 如何检查配置是否正确？

```bash
cd D:\动画\众生界
python -c "from core.config_loader import *; print(f'项目: {get_project_root()}'); print(f'模型: {get_model_path()}'); print(f'Qdrant: {get_qdrant_url()}')"
```

### Q: 配置不生效怎么办？

1. 检查文件名是否为 `config.json`（不是 `config.example.json`）
2. 检查JSON格式是否正确（可用在线JSON校验）
3. 重启Python进程（配置在启动时加载）

## 环境变量覆盖

配置可通过环境变量覆盖（优先级最高）：

| 环境变量 | 对应配置 |
|---------|---------|
| `NOVEL_PROJECT_ROOT` | `paths.project_root` |
| `NOVEL_CONFIG_PATH` | 配置文件路径 |
| `BGE_M3_MODEL_PATH` | `model.model_path` |
| `HF_HOME` | `model.hf_cache_dir` |

**示例**：
```bash
# Linux/Mac
export NOVEL_PROJECT_ROOT=/home/user/my-novel
python tools/build_all.py

# Windows PowerShell
$env:NOVEL_PROJECT_ROOT = "D:\我的小说"
python tools/build_all.py
```

## 完整配置示例

```json
{
  "project": {
    "name": "我的小说",
    "version": "1.0.0"
  },
  "paths": {
    "project_root": null
  },
  "database": {
    "qdrant_host": "localhost",
    "qdrant_port": 6333
  },
  "model": {
    "model_path": null
  },
  "novel_sources": {
    "directories": [
      "D:/我的小说资源库"
    ]
  }
}
```

**最简配置**：以上配置即可运行，其他项使用默认值。
